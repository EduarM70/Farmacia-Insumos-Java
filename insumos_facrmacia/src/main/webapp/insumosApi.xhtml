Web Services REST en Java (Medicinas)

Aprende sobre las Web Services REST y cómo funcionan en Java.

Las Web Services REST (Representational State Transfer) son un estilo arquitectónico para diseñar servicios web que utilizan los métodos estándar de HTTP (GET, POST, PUT, DELETE) para operar con recursos representados en formatos como JSON y XML.

¿Cómo funcionan las peticiones GET?

Las peticiones GET se utilizan para solicitar datos de un recurso específico en un servidor. Al realizar una petición GET, el servidor devuelve la representación del recurso solicitado en formato JSON o XML.

Información de la API

URL: /api/insumos

MÉTODO: GET

Utilidad: Muestra una lista de las medicinas o insumos médicos

Parámetros: Ninguno

Respuesta:

{
        "id": 1,
        "nombre": "Paracetamol",
        "categoria": "Antibióticos",
        "categoriaId": 0,
        "cantidad": 100,
        "precio": 0.5,
        "descripcion": "Analgésico y antipirético."
    }

Información de la API

URL: /api/insumos/{id}

MÉTODO: GET

Utilidad: Devuelve un insumo de acuerdo al ID ingresado

Parámetros:

  • Int: ID

Respuesta:

{
        "id": 1,
        "nombre": "Paracetamol",
        "categoria": "Antibióticos",
        "categoriaId": 0,
        "cantidad": 100,
        "precio": 0.5,
        "descripcion": "Analgésico y antipirético."
    }

Información de la API

URL: /api/categorias

MÉTODO: POST

Utilidad: Permite registrar un insumo nuevo en la base de datos

Parámetros:

  • String: Nombre
  • String: Descripción
  • Int: Cantidad
  • Int: Precio
  • Int: Categoria Id

Respuestas:

201 OK - El insumo fue creado exitosamente.

500 Internal Server Error - Ha ocurrido un problema con la solicitud.

Respuesta JSON:

{
        "id": 1,
        "nombre": "Nuevo insumo",
        "categoria": null,
        "categoriaId": 1,
        "cantidad": 100,
        "precio": 0.5,
        "descripcion": "Ejemplo de nuevo insumo"
    }

Información de la API

URL: /api/insumos/{id}

MÉTODO: POST

Utilidad: Permite actualizar los datos de un insumo

Parámetros:

  • Int: ID del insumo a editar
  • String: Nombre
  • String: Descripción
  • Int: Cantidad
  • Int: Precio
  • Int: Categoria Id

Respuestas:

201 OK - El insumo fue editado exitosamente.

500 Internal Server Error - Ha ocurrido un problema con la solicitud.

500 Internal Server Error - La categoría no existe en el registro.

Respuesta JSON:

{
        "id": 1,
        "nombre": "Insumo editado",
        "categoria": null,
        "categoriaId": 1,
        "cantidad": 100,
        "precio": 0.5,
        "descripcion": "Ejemplo de insumo actaulizado"
    }

Información de la API

URL: /api/insumos/delete/{id}

MÉTODO: DELETE

Utilidad: Elimina un insumo de la base de datos

Parámetros:

  • Int: ID del insumo a eliminar

Respuestas:

201 OK - El insumo fue eliminado exitosamente.

500 Internal Server Error - El insumo no existe.

Respuesta JSON:

{
        "id": 1,
        "nombre": "Paracetamol",
        "categoria": "Antibióticos",
        "categoriaId": 0,
        "cantidad": 100,
        "precio": 0.5,
        "descripcion": "Analgésico y antipirético."
